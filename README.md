### 🔷 ServiceHive — Full-Stack Lead Management Platform
> **React 19 · TypeScript · Node.js · Express.js 5 · MongoDB · JWT · Vite · Tailwind CSS**
> 🔗 [service-hive-nu.vercel.app](https://service-hive-nu.vercel.app)

My most complete full-stack project — built entirely solo. ServiceHive is a business lead management platform with a **strictly typed React 19 + TypeScript frontend** and a **production-hardened Express.js 5 + MongoDB backend**.

**What makes it technically solid:**
- 🔐 **End-to-end JWT authentication** — protected routes on both client and API, bcryptjs password hashing, Helmet + CORS security middleware on every endpoint
- 📋 **Full lead CRUD** — create, read, update, delete with server-side input validation via `express-validator`
- 🔍 **Custom debounce hook** for real-time search — no unnecessary API calls on every keystroke
- 📊 **Real-time activity dashboard** — live visibility into lead status and recent actions
- 🏗️ **Strict TypeScript throughout** — typed controllers, models, middleware, and API contracts on both layers. Not just frontend types — the entire codebase is typed.

### 🔷 Employee Payroll System — Backend-Focused Full-Stack App
> **Node.js · Express.js · EJS · HTML/CSS**
> 🔗 [employee-payroll-system-zm63.onrender.com](https://employee-payroll-system-zm63.onrender.com)

A full-stack payroll management application with non-trivial backend logic — not just CRUD.

**What's interesting under the hood:**
- 🧾 **Auto-billing middleware** that runs on every dashboard load — calculates net salary after 12% tax deduction, accumulates pending amounts per employee, and **prevents duplicate billing within the same calendar month** (checks if billing already ran this month before executing)
- ✅ **Date-conditional triggering** — billing only fires on or after the 5th of each month
- 🛡️ **Server-side validation** — regex email format checks, required-field enforcement, and duplicate-email guard before any write
- 📦 **Modular architecture** — all file I/O extracted into a dedicated `fileHandler` module; clean separation of concerns across the codebase
- 📊 **Dashboard aggregates** — total headcount, department count, gross payroll, and total pending obligations computed on the server

### 🔷 Real-Time Vehicle Counter — Computer Vision Pipeline
> **Python · YOLOv8 · OpenCV · SORT Multi-Object Tracker**

An applied computer vision system for real-world traffic monitoring — not a notebook demo, a working pipeline.

- 🎯 **YOLOv8m** with masked **region-of-interest inference** — detects only within the road zone, eliminating false positives from signs, buildings, and scene edges
- 🔁 **SORT multi-object tracker** (max_age=20, min_hits=3, IoU=0.3) — maintains stable vehicle IDs across frames even through occlusion and overlap
- ➕ **Centroid-crossing logic** — counts each unique vehicle exactly once as it crosses a defined line; zero double-counting
- 🖥️ Live count, bounding boxes, and custom UI overlaid directly on video output
